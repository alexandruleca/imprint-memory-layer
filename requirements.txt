# Canonical runtime deps live under requirements/ — split by profile so
# installers can pick what the user actually wants:
#
#   requirements/base.txt   — always installed (fastmcp, qdrant, onnxruntime CPU, extractors, FastAPI)
#   requirements/gpu.txt    — onnxruntime-gpu override (profile=gpu only)
#   requirements/llm.txt    — llama-cpp-python local tagger + chat (--with-llm only)
#
# This file is a dev convenience: `pip install -r requirements.txt` installs
# the full historical stack. Production installers use
# `imprint bootstrap --profile <cpu|gpu> [--with-llm]` instead, which shells
# out to the bundled `uv` and composes these files.
-r requirements/base.txt
-r requirements/llm.txt
